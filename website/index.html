Motivation

Virtual Reality immersion and throw detection

We combine immediate real time events with rich historical data to help answer the toughest questions about retention, growth, and engagement.

Goal

We poll for data on a millisecond basis. You can react to new information in seconds rather than days.

Learning objectives

We process our data across a massively distributed network of reliable servers to ensure 99.99% uptime.

Challenges

Throw detection is not a trivial task.

Other challenges we have faced:

  • Motion sensing in environments with interference from other people
  • Depth perception
  • Developing a game that is fun to play but doable in a tight timeframe
  • Immersion

Method

We combine several techniques to achieve an immersive experience

Body tracking. Microsoft Kinect is used for motion sensing input

VR Head Mounted Display. The game is displayed with Oculus Rift

Game Engine. Unity 3D and C# provide game logic and functionality

Conclusions

Playability is important

As the aim of the project was to make it as realistic as possible we focused a lot of time to make realistic 3D models for the tools, ingredients and the kitchen environment. We looked into bumpmapping and started to research how we would implement realistically looking fluids such as water, fire and oil.

For the interaction part we learned how to make use of both the Xbox 360 Kinect for skeleton tracking and the Oculus Rift for the virtual experience. We would have loved to use the Xbox One Kinect but the SDK for that Kinect requires Windows 8 or higher to work and since the Oculus Rift we used was attached to a computer which could not be updated to Windows 8 for various reasons we had to use the old Xbox 360 Kinect.
